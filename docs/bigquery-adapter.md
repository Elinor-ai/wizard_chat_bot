# BigQuery Adapter (Preview)

We now ship a lightweight adapter (`services/api-gateway/src/adapters/bigquery-adapter.js`) you can use to stream analytical data into Google BigQuery. It is not wired into any service yet—the goal is to make it easy to plug BigQuery in once the relevant collections outgrow Firestore.

## Environment Variables

Set the following keys (or pass the same values directly to `createBigQueryAdapter`):

| env key | required | description |
| --- | --- | --- |
| `BIGQUERY_PROJECT_ID` | ✅ | Target GCP project that owns the dataset. |
| `BIGQUERY_DATASET` | ✅ (for helper methods) | Default dataset you plan to use for inserts. |
| `BIGQUERY_TABLE` | ✅ (for helper methods) | Default table used for inserts. |
| `BIGQUERY_LOCATION` | optional | Dataset location (defaults to `"US"`). |
| `BIGQUERY_KEYFILE` | optional | Path to a service-account JSON file. Falls back to `GOOGLE_APPLICATION_CREDENTIALS`. |

> The adapter relies on `@google-cloud/bigquery`. Install it once in the repo root:  
> `npm install @google-cloud/bigquery`

## Usage

```js
import { createBigQueryAdapter } from "../adapters/bigquery-adapter.js";

const bigQuery = await createBigQueryAdapter({
  datasetId: "wizard_events",
  tableId: "llm_usage_snapshots"
});

await bigQuery.ensureDataset(); // creates dataset if needed
await bigQuery.ensureTable(
  [
    { name: "jobId", type: "STRING" },
    { name: "estimatedCostUsd", type: "FLOAT" }
  ]
);

await bigQuery.insertRows([
  { jobId: "job_123", estimatedCostUsd: 0.12 }
]);
```

Use the helper’s `insertRows` and `ensureTable` methods whenever you need to mirror a Firestore collection into BigQuery.
